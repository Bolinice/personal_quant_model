下面我直接给你一版**“模型管理界面原型结构”**，按你这 5 档套餐来设计，尽量做到：

- **一个统一界面**
- **按套餐逐级开放**
- **方便升级转化**
- **后续容易扩展**

我会按**页面结构 + 模块说明 + 权限映射**来写，你可以直接拿去给设计师/Figma 画原型。

---

# 一、整体结构

## 1. 页面入口结构

建议左侧导航这样设计：

```text
模型管理
├─ 总览
├─ 我的模型
├─ 模型模板
├─ 回测分析
├─ 调仓记录
├─ 数据导出
├─ API 接入
├─ 团队与权限
└─ 套餐与权益
```

其中：

- **基础版/进阶版/专业版**
  - 重点看：总览、我的模型、回测分析、调仓记录、套餐与权益
- **团队版**
  - 增加：团队与权限
- **机构版**
  - 增加：API 接入、组织级权限、定制入口

# 九、统一的权限交互原型

这是你整个系统最关键的一层。

---

## 1. 锁定状态设计

当用户没有权限时，不建议完全隐藏，建议这样：

### 按钮锁定态
```text
[导出全量数据 🔒]
hover 提示：专业版可解锁全量导出
```

### 模块锁定态
```text
因子暴露分析
当前套餐暂不支持该功能
升级专业版可解锁：因子暴露、归因分析、全A回测
[查看升级方案]
```

### 页面锁定态
```text
API 接入
当前套餐：进阶版
此功能为专业版及以上权益
[升级套餐] [联系顾问]
```

---

## 2. 权限标签统一样式

建议每个模型、页面头部都展示能力标签：

```text
[沪深300] [中证500] [Weekly] [部分导出] [个人版]
```

专业版模型可能是：

```text
[全A] [Daily] [API] [全量导出] [研究级]
```

团队版模型：

```text
[全部股票池] [全部频率] [团队共享] [高额度API]
```

# 十、推荐的页面流转

---

## 用户路径 1：普通用户查看模型
```text
总览 -> 我的模型 -> 点击模型 -> 回测结果 -> 发现高级功能锁定 -> 查看升级方案
```

## 用户路径 2：新用户创建模型
```text
总览 -> 模型模板 -> 选择模板 -> 创建模型 -> 进入详情 -> 回测分析
```

## 用户路径 3：团队管理员分配权限
```text
团队与权限 -> 添加成员 -> 分配角色 -> 指定可访问模型 -> 查看日志
```

## 用户路径 4：专业用户接 API
```text
API 接入 -> 创建 Key -> 查看文档 -> 调试接口 -> 查看调用日志
```
