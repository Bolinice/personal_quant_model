## 第1章 合规框架与商业模式

---

## 1. 概述

本文档规划量化策略平台从"个人研究工具"向"合规商业产品"的改造路径，核心目标：

1. **合规先行**：满足中国证监会、证券业协会对投资咨询/投顾服务的监管要求
2. **产品化交付**：从"策略代码"升级为"可订阅的标准化产品"
3. **风险隔离**：平台仅提供信息参考，不构成投资建议

---

## 2. 合规框架

### 2.1 适用法规

| 法规 | 核心要求 | 平台应对 |
|------|----------|----------|
| 《证券投资咨询管理暂行办法》 | 投资咨询需持牌 | 平台定位为"量化研究工具"，不提供投资建议 |
| 《证券期货投资咨询管理暂行办法》 | 禁止承诺收益 | 所有页面明确免责声明 |
| 《个人信息保护法》 | 用户数据保护 | 隐私政策 + 数据加密 + 最小化采集 |
| 《网络安全法》 | 数据安全 | HTTPS + 等保合规 |
| 《广告法》 | 禁止虚假宣传 | 营销文案审核机制 |

### 2.2 产品定位声明

> 本平台是**量化策略研究工具**，提供因子分析、策略回测、组合跟踪等功能。
> 平台输出的所有数据、信号、评分均为**历史统计结果**，不构成任何投资建议。
> 投资者应独立做出投资决策，自行承担投资风险。

### 2.3 合规红线

- **禁止承诺收益**：不得出现"保证收益"、"稳赚"等表述
- **禁止代客理财**：不提供账户代操作服务
- **禁止内幕信息**：所有数据来源公开市场
- **禁止虚假宣传**：回测结果必须标注"历史表现不代表未来"
- **用户适当性**：注册时风险测评，仅向合格用户开放

## 6. 数据安全与隐私

### 6.1 数据分级

| 级别 | 数据类型 | 保护措施 |
|------|----------|----------|
| L1（公开） | 因子说明、策略介绍 | 无特殊保护 |
| L2（内部） | 因子值、回测参数 | 登录可见 |
| L3（敏感） | 用户持仓、交易记录 | 加密存储 + 权限控制 |
| L4（核心） | API 密钥、支付信息 | 加密存储 + 审计日志 |

### 6.2 隐私保护措施

- **最小化采集**：仅采集必要信息
- **数据加密**：敏感字段 AES-256 加密
- **访问控制**：基于角色的数据访问
- **数据保留**：用户注销后 30 天内删除个人数据
- **审计日志**：所有敏感操作记录审计日志

## 5. 交互设计

### 5.1 关键流程

**回测流程**：
```
选择策略 → 配置参数 → 确认(含合规提示) → 执行(进度条) → 结果展示(含免责)
```

**订阅流程**：
```
选择方案 → 风险测评(未测评用户) → 确认订阅 → 支付 → 开通权限
```

**信号推送**：
```
选择策略 → 配置推送规则 → 确认(含合规提示) → 接收推送(每条含免责)
```

### 5.2 加载与反馈

- **骨架屏**：所有数据页面使用骨架屏加载
- **进度条**：回测等长时任务显示进度
- **空状态**：无数据时展示引导文案
- **错误状态**：友好的错误提示 + 重试按钮
