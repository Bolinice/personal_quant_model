# 滚动因子筛选（Rolling Factor Selection）

滚动因子筛选是一种动态评估与挑选有效因子的方法。它的核心思想是：**不依赖全样本历史数据做一次性筛选，而是沿着时间轴按固定窗口逐步向前移动，定期重新评估因子表现，并据此更新模型中使用的因子集合。**

下面从原理、流程、优缺点及注意事项四个维度展开说明：

## 一、为什么需要“滚动”？
金融市场具有**非平稳性（Non-stationarity）**：
- 宏观周期、监管政策、投资者结构、交易机制的变化会导致因子有效性发生衰减、反转或阶段性失效（如2017年“漂亮50”行情中价值因子占优，2020年后小盘成长因子重新活跃）。
- 若用全样本历史做静态筛选，容易过拟合特定时期，样本外表现大幅下滑。
- 滚动筛选通过**“局部历史 + 动态更新”**，使模型能 切换，提升稳健性。

## 二、操作流程
1. **设定参数**
   - 滚动窗口长度（如 24 个月、36 个月）
   - 滚动步长/频率（如每月末、每季度末更新）
2. **窗口内评估**
   - 在每个时间窗口内，对候选因子计算预测能力指标（如 IC、ICIR、Rank IC、多空收益等）。
3. **筛选规则**
   - 按阈值（如 ICIR > 0.5）或排序（取 Top N）选出当期有效因子。
4. **应用与更新**
   - 将筛选结果用于下一期组合构建或因子加权；窗口向前滚动一步，重复流程。
5. **自动化流水线**
   - 嵌入量化平台：`因子计算` → `截面/时序检验` → `动态筛选` → `组合优化` → `回测验证`。

## 三、优缺点对比
| 优点 | 缺点 |
| :--- | :--- |
| 适应市场结构变化，降低过拟合风险 | 计算成本高，需维护因子库与自动化流水线 |
| 可捕捉因子生命周期（孕育 → 有效 → 衰减） | 窗口过短易受噪声干扰，过长则滞后 |
| 样本外表现通常更稳定 | 频繁调仓可能增加交易成本与滑点 |
| 便于与因子择时、动态权重结合 | “追涨杀跌”式误判 |

> 💡 **注**：评估前通常需做市值/行业中性化、极值处理、缺失值填充等预处理，避免伪相关。

## 四、注意事项
1. **窗口长度选择**：需平衡“统计显著性”与“时效性”。股票因子常用 2~3 年，高频/量价因子可能用 3~6 个月。
2. **滚动频率控制**：不宜过高（如每日滚动），通常月度或季度更新即可，避免过度交易。
3. **防未来函数**：严格对齐数据时间戳，确保评估期仅使用当期及历史信息。
4. **结合经济逻辑**：纯数据挖掘易陷入“因子动物园”陷阱，需要用宏观/微观逻辑过滤候选池。
5. **平滑与阈值**：可加入移动平均、IC 衰减加权、或设置“连续 N 期达标才纳入”等规则，降低噪声干扰。
6. **与机器学习结合**：滚动筛选后可接动态权重分配（如 Ridge/Lasso 回归、因子暴露约束、强化学习择时）。
