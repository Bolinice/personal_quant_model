## 1. 平台概述

A股多因子增强策略平台是一个专业的量化投资策略研究与管理平台，提供基于多因子选股模型的组合建议和调仓信号服务。

**平台解决三个核心问题**：
1. **买什么股票** — 从A股里挑出更可能跑赢基准的股票
2. **买多少** — 把"看好的股票"变成满足风险约束的实际持仓权重
3. **什么时候多买、少买** — 控制整体仓位，避免在很差的市场环境下满仓硬扛

### 1.1 核心功能
- **策略订阅**：订阅专业量化策略，获取组合建议
- **组合管理**：查看当前推荐组合和调仓记录
- **绩效跟踪**：实时监控策略表现和收益情况
- **风险提示**：了解策略风险和适用范围

### 1.2 策略特点
- 质量成长+预期修正+残差动量的行业约束增强策略，重视回撤控制
- 透明可解释的选股逻辑，ML增强可回退
- 适合中低频投资策略

### 1.3 V2基线量化目标
- 年化超额收益 > 8%（成本后）
- Sharpe > 1.0
- 最大回撤 < 20-25%
- 年化换手 2-6 倍
- 基准：中证800

### 1.4 四大Alpha模块通俗解释

| 模块 | 一句话解释 | 预测周期 |
|------|-----------|---------|
| 价格行为 | 看K线、动量、波动率，捕捉短期价格趋势和反转 | 5-10日 |
| 价值质量 | 找便宜的好公司——估值低+盈利质量高 | 10-20日 |
| 预期修正 | 跟踪分析师和盈利惊喜，预期上调的股票往往继续涨 | 5-20日 |
| 资金流 | 跟踪聪明钱（北向资金、大单），资金先动价格后动 | 3-10日 |

### 1.5 择时信号通俗解释
择时决定"现在该不该多买"。当市场环境好（趋势向上、波动率低、大多数股票在涨），策略会提高仓位；当市场环境差（高波动、大跌、流动性收缩），策略会自动降低仓位保护本金。您可能会看到仓位在30%-100%之间变化，这是正常的防御机制。

- **适用说明**：了解策略的适用场景和风险提示

### 4.2 股票池参数
策略选股范围基于以下过滤条件：

| 过滤维度 | 参数值 | 说明 |
|----------|--------|------|
| 上市天数 | ≥ 180天 | 排除次新股波动风险 |
| 日均成交额 | ≥ 8000万 | 确保流动性充足 |
| 总市值 | ≥ 80亿 | 排除小市值流动性风险 |
| 股价 | ≥ 3元 | 排除仙股和退市风险股 |
| 风险事件过滤 | 是 | 排除重大风险事件个股 |
| 黑名单硬过滤 | 是 | 排除黑名单中的个股 |

### 4.3 因子模块体系
策略采用5模块+1风险模块的架构：

| 模块名称 | 权重 | 核心逻辑 |
|----------|------|----------|
| 质量成长模块 | 35% | ROE稳定性、盈利增长质量、现金流健康度 |
| 预期修正模块 | 30% | 分析师预期修正、盈利惊喜、预期变化动量 |
| 残差动量模块 | 25% | 行业中性残差动量、动量衰减特征 |
| 资金流确认模块 | 10% | 主力资金流入、北向资金确认 |
| 风险惩罚模块 | 独立扣分（λ=0.35） | 波动率惩罚、尾部风险惩罚、流动性风险惩罚 |

### 4.4 择时机制
策略采用风险预算择时机制：
- **5信号4档仓位**：30% / 60% / 80% / 100%
- **回撤保护规则**：
  - 回撤 > 5%：仓位降一档
  - 回撤 > 8%：仓位再降一档
  - 回撤 > 12%：仓位降至最低30%

### 4.5 组合构建
策略采用分层赋权与风险折扣机制：

**分层赋权（Top60）**：
- 第1-10名：1.5x 基础权重
- 第11-30名：1.2x 基础权重
- 第31-60名：1.0x 基础权重

**风险折扣**：
- 低风险：1.00（无折扣）
- 中风险：0.85
- 高风险：0.60

**约束规则**：
- 单票上限：3%
- 行业偏离：±5%

## 8. 常见问题

### 8.1 如何理解调仓信号？
调仓信号是基于多因子模型计算的结果，包含：
- 买入建议：推荐增加仓位的股票
- 卖出建议：推荐减少仓位的股票
- 仓位调整：根据市场情况调整整体仓位

### 8.2 如何评估策略表现？
通过以下指标评估：
- 年化收益率：衡量长期收益
- 最大回撤：衡量风险控制能力
- 信息比率：衡量超额收益的稳定性
- 夏普比率：衡量风险调整后的收益

### 8.3 策略适合什么样的投资者？
适合：
- 长期投资者
- 风险承受能力中等的投资者
- 希望获得稳健超额收益的投资者

不适合：
- 短期交易者
- 风险承受能力低的投资者
- 追求极端收益的投资者

### 8.4 如何处理调仓建议？
建议：
- 在下一个交易日开盘时执行调仓
- 考虑自身持仓情况和交易成本
- 可根据个人情况适当调整

## 9. 风险控制说明

### 9.1 三层风控体系
策略采用三层风控体系，确保组合风险可控：

| 风控层级 | 覆盖范围 | 主要功能 |
|----------|----------|----------|
| 准入风控 | 股票池过滤 | 排除ST/退市/流动性不足/黑名单个股 |
| 持仓风控 | 个股监控 | 风险惩罚扣分、风险折扣降权 |
| 组合风控 | 整体组合 | 行业偏离约束、集中度约束、回撤保护 |

### 9.2 黑名单系统
- **监管类黑名单**：被证监会立案调查、重大违规的个股
- **流动性黑名单**：连续多日成交额低于阈值的个股
- **事件类黑名单**：重大负面事件（财务造假、重大诉讼等）
- **自动更新**：黑名单每日自动检查和更新

### 9.3 风险惩罚规则引擎
风险惩罚模块独立于评分模块运作，直接对综合评分进行扣分：
- 波动率惩罚：过高波动率个股扣分
- 尾部风险惩罚：极端下行风险个股扣分
- 流动性风险惩罚：流动性恶化个股扣分
- 惩罚强度由参数 λ=0.35 控制

### 9.4 极端情景保护
- **暴跌保护**：单日跌幅超阈值时自动触发风控
- **系统性风险**：市场整体风险升高时自动降仓
- **流动性危机**：市场流动性急剧收缩时暂停调仓

## 10. ML增强说明

### 10.1 三路融合架构
策略采用三路融合的评分机制：

| 信号来源 | 权重 | 说明 |
|----------|------|------|
| 规则信号 | 0.30 | 基于金融逻辑的规则评分，稳定可解释 |
| Ridge回归 | 0.25 | 线性模型，捕捉因子线性关系 |
| LightGBM | 0.45 | 非线性模型，捕捉因子交互效应 |

### 10.2 回退机制
- ML模型在以下情况自动切换为规则基线：
  - 模型预测IC连续低于阈值
  - 模型输出分布显著漂移
  - 关键特征缺失导致模型无法正常运行
- 回退时权重调整：规则信号权重提升至1.0，ML信号权重归零
- 回退状态自动监控，模型恢复后自动切回融合模式

## 11. 因子监控与告警说明

### 11.1 全链路5类监控

| 监控类型 | 监控内容 | 告警阈值 |
|----------|----------|----------|
| 数据监控 | 数据源可用性、缺失率、延迟 | 缺失率>5%或延迟>30min |
| 因子监控 | IC漂移、PSI稳定性、KS检验 | IC<0.02或PSI>0.25 |
| 模型监控 | 预测IC、预测分布、特征重要性 | IC<0.01或分布偏移>2σ |
| 组合监控 | 行业偏离、集中度、换手率 | 偏离>3%或换手>8倍 |
| 实盘监控 | 滑点、执行偏差、成交率 | 滑点>5bp或执行偏差>10bp |

### 11.2 自动响应机制
- **轻度告警**：通知运维人员，记录日志
- **中度告警**：自动切换至保守参数，通知策略团队
- **重度告警**：自动暂停策略信号输出，等待人工确认

### 11.3 模块状态机
每个因子模块具有以下状态：

| 状态 | 说明 | 行为 |
|------|------|------|
| active | 正常运行 | 满权重参与评分 |
| watch | 观察期 | 正常参与，但标记为需关注 |
| degraded | 降级运行 | 权重降低，发出告警 |
| disabled | 已禁用 | 不参与评分，权重重新分配 |

状态转换条件：
- active → watch：IC连续3期低于观察阈值
- watch → degraded：IC连续5期低于降级阈值或PSI超限
- degraded → disabled：IC持续恶化或数据源中断
- watch → active：IC恢复至正常水平
- degraded → watch：IC连续3期回升
- disabled → degraded：数据源恢复，IC回升至降级阈值以上
