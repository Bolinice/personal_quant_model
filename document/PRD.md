# 1. 项目背景

## 1.1 行业背景
A股市场具备以下特征：

- T+1交易制度
- 涨跌停限制
- 停牌、复牌事件频繁
- 财务因子、估值因子、质量因子和风格因子长期具备研究价值
- 相比超短线与高频，中低频量化更适合个人和中小团队落地
- 市场风格切换快，指数增强与多因子增强具备较强商业表达能力

## 1.2 业务背景
当前量化产品普遍存在以下问题：

- 因子研究流程不规范，容易出现未来函数
- 回测表现优异但实际不可交易
- 组合约束和风控设计薄弱
- 研究、模拟、交付、订阅各环节割裂
- 客户侧只收到信号，缺乏透明解释和持续跟踪能力
- 缺乏事件风控和个股风险排雷能力
- 因子模块缺乏状态管理机制
- 组合构建过度依赖精细优化器，实盘鲁棒性不足

##1.3 建设必要性
因此，需要建设一套面向 A 股中低频量化的标准化平台，支持：

- 股票池管理
- 多因子研究
- 多因子组合构建
- 指数择时与仓位管理
- 风险控制与调仓管理
- 事件风控与风险惩罚管理
- 数据快照与版本管理
- 因子研究标准流程与上线闸门
- 回测与模拟验证
- 商业化交付与订阅管理

该平台既服务于内部研究和运营，也支持对外策略产品化交付。

# 2. 产品定义

## 2.1 产品名称
A股多因子增强策略平台

## 2.2 产品定位
本产品是一套面向 A 股市场的**中低频量化研究与策略交付平台**，以：

- 多因子选股
- 指数择时
- 风险控制
- 组合构建
- 模拟跟踪
- 信号订阅与报告交付

为核心能力，目标是构建从研究到交付的完整闭环。

V2策略定位说明：V2主策略定位为A股中低频、偏质量成长+预期修正+残差动量的行业约束增强策略，重视回撤控制。

## 2.3 产品价值
平台核心价值包括：

1. 为研究员提供标准化多因子研究与回测环境  
2. 为组合经理提供可执行、可解释、可约束的组合构建工具  
3. 为风控提供风险暴露、回撤、流动性和模型失效监控能力  
4. 为客户提供清晰的组合建议、调仓记录、净值表现和风险提示  
5. 为业务提供可订阅、可交付、可持续迭代的量化策略产品能力

# 3. 产品目标

## 3.1 业务目标
- 构建至少 1 套稳定运行的 A 股多因子增强模型
- 支持至少 1 种标准商业化交付方式：组合建议 / 调仓信号订阅
- 支持至少 100 名客户的订阅管理
- 建立标准化策略报告输出机制

## 3.2 策略目标
在控制风险和回撤前提下，争取实现：

- 长期跑赢沪深300 / 中证500 / 中证1000 / 中证全指等基准
- 超额收益长期为正
- 换手率可控
- 风格暴露可控
- 回撤优于高波动单风格策略
- 模拟与回测偏差可解释

V2基线量化目标：
- 年化超额收益 > 8%（成本后）
- Sharpe比率 > 1.0
- 最大回撤 < 20-25%
- 年化换手率控制在2-6倍区间
- 基准：中证800

## 3.3 平台目标
平台需支持：

- 研究—回测—模拟—发布—交付—复盘完整闭环
- 所有关键数据支持点时回溯
- 回测结果可复现
-仓输出可追踪、可审计、可回滚
- 所有因子必须经过标准化研究流程才能上线
- 模型输出必须有回退机制（规则基线fallback）
- 配置驱动，参数不写死代码

# 7. 业务流程

## 7.1 策略研发流程
1. 研究员选择基准、样本区间和股票池  
2. 选择或定义因子集合  
3. 平台完成因子预处理与中性化  
4. 输出单因子分析结果  
5. 配置多因子权重方案  
6. 生成历史组合并回测  
7. 评估收益、超额、回撤、换手和暴露  
8. 通过评审后进入模拟组合  
9. 模拟通过后正式发布  

## 7.2 客户交付流程
1. 客户注册并订阅策略  
2. 平台校验订阅权限  
3. 按调仓周期发布组合建议和信号  
4. 客户查看组合、调仓、净值和风险提示  
5. 平台持续输出报告和提醒  
6. 客户续费或到期停用  

## 7.3 风控运行流程
1. 数据任务更新  
2. 因子任务运行  
3. 模型任务打分  
4. 风控任务校验  
5. 调仓任务输出  
6. 报告任务生成  
7. 告警任务监控  
8. 异常策略进入人工复核

# 9. 详细需求说明

---

## 9.1 模块一：A股数据管理

### 9.1.1 目标
提供稳定、可回溯、可审计的 A 股基础数据服务。

### 9.1.2 功能需求
- 支持日线行情数据管理
- 支持分钟线行情扩展
- 支持复权因子管理
- 支持财务数据及发布日期管理
- 支持指数成分股历史管理
- 支持停牌、涨跌停、ST、退市等状态管理
- 支持行业分类和历史变更追踪
- 支持交易日历和公司行为事件
- 支持数据质量检查（OHLC一致性、涨跌幅校验、覆盖率、交叉验证）
- 支持PIT（Point-In-Time）前视偏差防护，财务数据按公告日使用
- 支持幸存者偏差处理，回测时包含退市股票避免收益虚高
- 支持事件数据中心（业绩预告/问询函/立案处罚/减持/股权质押/回购/解禁等）
- 支持数据快照与版本管理（每日生成snapshot_id，回测绑定快照版本）
- 支持PIT财务数据严格版本管理（预告/快报/正式报表优先级、修订版本号）
- 支持分析师与北向资金PIT表

### 9.1.3 关键规则
- 财务数据必须按公告发布日期使用，禁止未来函数
- 股票状态需支持按交易日回放
- 数据缺失与异常需可监控
- 数据口径变更需留痕
- 回测股票池必须包含历史退市股票，避免幸存者偏差

### 9.1.4 验收标准
- 任意交易日可还原股票池状态
- 任意股票可查询点时历史信息
- 不允许未来函数穿越

---

## 9.2 模块二：股票池管理

### 9.2.1 目标
构建可投资、可回放、可解释的股票池。

### 9.2.2 功能需求
支持以下股票池类型：
- 沪深300
- 中证500
- 中证1000
- 中证全指
- 自定义股票池

以下过滤规则：
- 剔除 ST / *ST
- 剔除停牌
- 剔除上市未满 N 天（默认180个交易日）
- 剔除日均成交额过低（默认8000万）
- 剔除极端低价股（默认3元以下）
- 剔除涨跌停无法成交股票
- 剔除财务异常股票
- 剔除退市整理股票
- 剔除重大风险事件股（近60日无重大立案/处罚/严重审计问题）
- 黑名单硬过滤（ST/退市整理/重大立案/严重审计非标）

支持扩展筛选：
- 市值区间（核心池总市值≥80亿）
- 行业范围
- 板块范围
- 波动率范围
- 北向资金偏好
- 估值边界

### 9.2.3 验收标准
- 可按日期重建股票池
- 可股票池模板
- 可展示过滤前后数量变化
- 可展示个股被剔除原因

## 9.3 模块三：因子库与因子分析

### 9.3.1 目标
建立标准化因子体系和因子评估机制。

### 9.3.2 因子范围
支持以下5模块+1风险模块：

**质量成长模块**
- ROE_TTM / ROA_TTM / 毛利率 / CFO / 净利润比 / 营收增速 / 净利润增速 / Sloan应计

**预期修正模块**
- SUE / 盈利惊喜 / 分析师修正 / 覆盖度变化 / PEAD

**残差动量模块**
- 12月动量跳1月 / 6月动量跳1月 / 残差动量 / 1月反转 / 低波动率

**资金流确认模块**
- 北向净买入 / 换手率趋势 / Amihud非流动性

**风险惩罚模块**
- 商誉占比 / 股权质押 / 业绩预告下修 / 审计异常 / 问询立案 / 现金流异常 / 减持

### 9.3.3 功能需求
- 支持因子定义与管理
- 支持因子计算任务
- 支持缺失处理、去极值、标准化
- 支持行业中性化、市值中性化
- 支持因子方向统一
- 支持因子覆盖率和稳定性筛选

支持因子元数据系统：
- 因子身份证（名称/逻辑/公式/来源/PIT/方向/覆盖率/状态/版本）

支持因子预处理统一流水线（10步固定流程）：
1. 对齐 → 2. 缺失标记 → 3. 覆盖率检查 → 4. 填充 → 5. 去极值 → 6. 中性化 → 7. 标准化 → 8. 方向统一 → 9. 正交化 → 10. 存快照

支持因子研究标准7关：
1. 定义关 → 2. 数据关 → 3. 统计关 → 4. 成本关 → 5. 去重关 → 6. 稳健性关 → 7. 上线关

支持因子上线闸门：
- 覆盖率 > 70%
- RankIC均值 > 0
- 正IC比例 > 52%
- 分年度不极端
- 成本后有边际价值

支持分析能力：
- IC / RankIC
- 分层回测
- 多空分层
- 衰减分析
- 相关性分析
- 覆盖率分析
- 分年度分析
- 不同风格阶段分析

### 9.3.4 验收标准
- 可查看单因子历史表现
- 可输出图表和分析报告
- 可查看因子矩阵和相关性矩阵

## 9.4 模块四：多因子模型构建

### 9.4.1 目标
支持多因子组合形成综合评分模型，并提供版本化管理。

### 9.4.2 功能需求
支持模型方式：
- 等权加权
- 人工权重
- IC / IR 动态加权
- 分层筛选
- 排序打分
- 动态权重收缩（w = 0.7*w_base + 0.3*w_dynamic_shrunk，限制单模块权重10%-45%）
- ML增强排序（两阶段：模块内聚合→ML融合层）
- 三路融合方案（规则0.30 + Ridge0.25 + LightGBM0.45）

支持模块状态机：
- active（正常参与加权）
- watch（监控中，权重受限）
- degraded（降级，权重大幅收缩）
- disabled（不参与计算）

V2基线模块权重：
- 质量成长 35%
- 预期修正 30%
- 残差动量 25%
- 资金流确认 10%
- 风险惩罚系数λ=0.35，独立扣分不参与正向加权

支持预测置信度体系：
- 模块一致性
- 特征缺失
- 模型稳定性

支持模型注册表：
- model_id / feature_set / label_version / snapshot_id / params / oof_metrics / status

支持模型输入：
- 股票池
- 因子列表
- 权重方案
- 调仓频率
- 持仓数
- 中性化设置
- 风险约束
 基准

支持模型输出：
- 综合得分
- 因子得分明细
- 排名结果

# 19. 附录

## 19.1 术语说明
- 多因子：基于多个量化因子综合评分进行选股的方法
- IC：因子值与未来收益的相关性指标
- RankIC：因子排序与未来收益排序的相关性指标
- 中性化：剔除行业、市值等非目标暴露影响
- 择时：根据市场状态调整仓位
- 超额收益：策略收益减去基准收益
- 调仓：根据模型结果更新持仓组合

## 19.2 文档结论
本 PRD 定义了一套可用于 A 股中低频量化研究、策略交付和客户订阅的正式产品方案。
首期重点不在追求复杂模型，而在于建设：

- 标准化的数据与因子体系
- 可落地的组合构建体系
- 真实化的回测与模拟体系
- 稳健的风控与交付体系
- 清晰的商业化订阅能力

V2升级重点不在堆更多功能，而在三件事：做减法（主策略收敛最稳版本）、做深度（每个因子/模块/风险源更严谨）、做差异化（事件风控/预期差/风险排雷）。
